# /api — the entire LazyTools backend

Two PHP files. That's the whole server side of LazyTools, and it exists only for **anonymous tool
ratings**. Tools themselves never talk to any server (see [/how-it-works/](https://lazytools.io/how-it-works/)).

| File | Purpose |
|---|---|
| `rate.php` | `POST {"tool", "stars"}` → one row. Stores tool slug + stars + timestamp. **No IP, no cookies, no UA, no identifiers.** |
| `stats.php` | `GET` → aggregate `{tool: {count, avg}}`, only for tools with ≥25 ratings (the display threshold). |
| `schema.sql` | The one-table schema. |
| `config.sample.php` | Copy to `config.php` on the server with cPanel MySQL credentials. Gitignored. |
| `tools-allowlist.json` | Regenerated by `npm run build` from the tool registry — `rate.php` rejects slugs that aren't real tools. |

## Deploy (Namecheap cPanel)

1. cPanel → MySQL Databases: create DB + user, grant ALL, run `schema.sql` in phpMyAdmin.
2. Upload `api/` to `public_html/api/`, copy `config.sample.php` → `config.php`, fill credentials.
3. Verify: `curl -X POST https://lazytools.io/api/rate.php -d '{"tool":"units/kg-to-lbs","stars":5}'`
   → `{"ok":true}`; `curl https://lazytools.io/api/stats.php` → `{}` until thresholds are met.

Abuse posture: the localStorage flag client-side softens duplicate votes; the ≥25 display threshold
makes low-effort manipulation pointless. We deliberately accept some noise rather than fingerprint users.
